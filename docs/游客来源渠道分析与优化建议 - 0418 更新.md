# 游客来源渠道分析与优化建议（0418 更新）

> 数据更新时间：2026 年 4 月 18 日（最新数据）
> 数据来源：`visitor_source_records` + `game_sessions` + `player_statistics` 表实际统计
> 分析周期：2026-04-15 至 2026-04-18（共 4 天）
> 
> **转化率定义说明：**
> - **开始游戏转化率** = 有 game_sessions 记录的玩家 / 游客总数（只要做过一次抽卡操作就算转化）
> - **完成游戏转化率** = 有 player_statistics 记录（total_games > 0）的玩家 / 游客总数（完成一局完整游戏才算转化）
> - **中途流失率** = (开始游戏玩家 - 完成游戏玩家) / 开始游戏玩家

## 3. 关键问题分析

### 3.1 转化漏斗分析

```
总游客数：1,152 人
    ↓ 11.72% 转化
开始游戏：135 人 (有 game_sessions 记录)
    ↓ 25.19% 留存
完成对局：34 人 (有 player_statistics 记录，total_games > 0)
```

**关键流失点：**
1. **首页 → 开始游戏**：流失 1,017 人（88.28%），主要问题：
   - 广告渠道（1069/1274）用户被动跳转，游戏意愿低
   - 首页缺乏有效的"立即试玩"引导
   - 广告素材与游戏实际内容不符，用户预期落差大

2. **开始游戏 → 完成对局**：流失 101 人（74.81%），可能原因：
   - 游戏难度过高，新手无法完成首局
   - 选卡界面复杂，用户放弃
   - 广告渠道用户耐心不足，轻易放弃

### 4.2 具体优化方案

#### 方案 A：新游客"立即试玩"引导（优先级 P0）
**适用场景**：所有广告渠道（1069/1274/1095）

```typescript
// 在 GameManager.ts 的 updateHomeKillRankLabel 完成后
if (this.lastKillRankKills === 0 || !this.hasKillRankCache) {
    // 新游客（杀敌数为 0），显示"立即试玩"按钮 + 首局奖励提示
    this.showTryNowButton();

// 针对广告渠道用户
const visitorScene = this.lastVisitorScene || 'unknown';
if (visitorScene.includes('1069') || visitorScene.includes('1274') || visitorScene.includes('1095')) {
    this.showFirstGameBonusTip(); // 首局通关奖励提示
}
```

#### 方案 B：优量汇广告（1274）紧急优化（优先级 P0）
**问题**：133 人 0% 完成转化，广告素材与游戏严重不符

**建议措施：**
1. 检查广告素材是否过度承诺（如"一刀 999"类夸张素材）
2. 调整广告落地页，增加游戏实际画面展示
3. 新手引导简化，前 3 波自动防御，降低上手门槛

#### 方案 C：unknown 渠道异常排查（优先级 P0）
**问题**：10 人人均 510 次访问，总访问 63,801 次，疑似刷量或测试行为

**建议措施：**
1. 检查是否有自动化测试脚本未过滤
2. 排查数据采集逻辑是否重复上报
3. 如确认异常，应在统计中剔除

#### 方案 D：nearby 渠道保持优化（优先级 P1）
**优势**：54.05% 开始转化率，18.92% 完成转化率，均为最高水平

**建议措施：**
1. 保持现状，不过度干预
2. 可考虑增加 LBS 特色玩法（如"附近玩家排行榜"）
3. 适当增加附近小程序曝光投入

## 6. 数据监控建议

### 6.1 每日监控指标

```sql
-- 每日游客来源统计
SELECT 
    DATE(FROM_UNIXTIME(last_access_time / 1000)) AS access_date,
    channel,
    COUNT(DISTINCT player_id) AS unique_visitors,
    SUM(access_count) AS total_visits
FROM visitor_source_records
WHERE last_access_time >= UNIX_TIMESTAMP(CURDATE() - INTERVAL 7 DAY) * 1000
GROUP BY DATE(FROM_UNIXTIME(last_access_time / 1000)), channel
ORDER BY access_date DESC, unique_visitors DESC;
```

### 6.2 异常告警阈值

| 指标 | 正常范围 | 告警阈值 | 应对措施 |
|------|---------|---------|---------|
| 单日游客数 | 200-400 人 | >500 人或<100 人 | 检查渠道投放/数据采集 |
| 人均访问次数 | 2-5 次 | >50 次 | 排查刷量/测试行为 |
| 开始转化率 | 10-20% | <5% 或>80% | 检查广告素材/数据采集 |
| 完成转化率 | 2-10% | <1% | 检查游戏难度/服务器稳定性 |

## 8. 总结

1. **4 月 15 日广告投放效果显著但质量堪忧**：游客量从 5 人激增至 1,152 人，但开始转化率仅 11.72%，完成转化率仅 2.95%

2. **三大广告渠道问题突出**：
   - scene_1069（613 人）：5.06% 开始转化率，0.49% 完成转化率
   - scene_1274（133 人）：4.51% 开始转化率，0% 完成转化率
   - scene_1095（320 人）：14.69% 开始转化率，2.81% 完成转化率

3. **unknown 渠道数据异常**：10 人人均 510 次访问，需排查是否刷量或测试行为

4. **nearby 渠道表现优异**：54.05% 开始转化率，18.92% 完成转化率，可适当增加投入

5. **核心优化方向**：
   - 简化广告渠道用户落地流程
   - 降低新手难度，提升首局完成率
   - 优化广告素材与游戏一致性
   - 排查异常数据，确保统计准确性
