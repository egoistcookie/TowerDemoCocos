# 游客来源渠道分析与优化建议

> 数据更新时间：2026 年 4 月 15 日（最新数据）
> 数据来源：`visitor_source_records` + `game_sessions` + `player_statistics` 表实际统计
> 
> **转化率定义说明：**
> - **开始游戏转化率** = 有 game_sessions 记录的玩家 / 游客总数（只要做过一次抽卡操作就算转化）
> - **完成游戏转化率** = 有 player_statistics 记录的玩家 / 游客总数（完成一局完整游戏才算转化）
> - 之前的分析使用 player_statistics，导致转化率被低估（15.19%），实际开始游戏转化率为 24.05%

## 1. 场景值 1256 含义

**场景值 1256** = **PC 端微信小游戏面板「最近使用」列表**

### 详细解释
- 用户通过 **PC 版微信** 的小程序面板中的「最近使用」列表打开小游戏
- 这是 PC 端特有的入口，属于**自然回访**行为
- 同类 PC 端场景值：
  | 场景值 | 说明 |
  |--------|------|
  | 1256 | PC 端小程序面板「最近使用」列表 |
  | 1257 | PC 端小程序面板「我的小程序」 |
  | 1258 | PC 端小程序面板其他入口 |

## 4. 优化建议：提升开始游戏转化率

### 4.1 当前问题分析

根据转化漏斗数据：
- **75.95% 的游客停留在首页，未开始游戏**（60/79 人）
- **36.84% 的开始玩家未完成对局**（7/19 人）

**核心问题：**
1. 首页缺乏明显的"开始游戏"引导
2. 广告渠道（1274）用户点击广告后未开始游戏
3. 移动应用渠道（1069）落地体验差
4. 游戏难度可能过高，新手无法完成首局

### 4.2 优化方案

#### 方案 A：新游客增加"立即试玩"引导（优先级 P0）
**适用场景**：所有渠道，尤其是 scene_1069（H5 广告跳转）、scene_1274、nearby

```typescript
// 在 updateHomeKillRankLabel 完成后，添加"立即试玩"引导
if (this.lastKillRankKills === 0 || !this.hasKillRankCache) {
    // 新游客（杀敌数为 0），显示"立即试玩"按钮 + 首局奖励提示
    this.showTryNowButton();
}
```

#### 方案 B：针对回访渠道（1256/1027/1028）优化（优先级 P1）
**适用场景**：PC 端回访、历史记录、我的小程序

```typescript
// 检测到回访渠道时，自动弹出上局战绩或快捷开始
if (wxLaunchOptions.scene === 1256 || wxLaunchOptions.scene === 1027 || wxLaunchOptions.scene === 1028) {
    // PC 端回访用户，直接显示"继续游戏"或"快速开始"
    this.showQuickStartForReturningVisitor();
}
```

#### 方案 C：广告渠道特殊处理（优先级 P0）
**适用场景**：scene_1274（优量汇广告）、scene_1095（小程序广告组件）

```typescript
// 广告渠道用户（1274 优量汇、1095 广告组件），增加新手引导和首局奖励
if (wxLaunchOptions.scene === 1274 || wxLaunchOptions.scene === 1095) {
    // 显示新手引导，首局通关奖励提示
    this.showNewbieGuideAndFirstGameBonus();
}
```

#### 方案 D：移动应用渠道优化（优先级 P0）
**适用场景**：scene_1069（最大游客来源，105 人，转化率仅 4.76%）※ 实际为 H5 广告跳转

```typescript
// scene_1069 是从 H5 广告跳转/外部应用的用户，需要简化落地流程
if (wxLaunchOptions.scene === 1069) {
    // 跳过复杂引导，直接进入"快速开始"流程
    this.showQuickStartForExternalAppVisitor();
}
```

### 5.2 渠道转化率 SQL 查询

```sql
-- 各渠道转化率分析（从游客到玩家的转化）
-- 使用 game_sessions 表判断是否开始游戏（只要做过一次抽卡操作就算）
SELECT
    v.channel,
    v.scene,
    COUNT(DISTINCT v.player_id) AS total_visitors,
    COUNT(DISTINCT gs.player_id) AS started_game,
    COUNT(DISTINCT ps.player_id) AS completed_game,
    ROUND(COUNT(DISTINCT gs.player_id) * 100.0 / COUNT(DISTINCT v.player_id), 2) AS start_rate,
    ROUND(COUNT(DISTINCT ps.player_id) * 100.0 / COUNT(DISTINCT v.player_id), 2) AS complete_rate,
    ROUND((COUNT(DISTINCT gs.player_id) - COUNT(DISTINCT ps.player_id)) * 100.0 / NULLIF(COUNT(DISTINCT gs.player_id), 0), 2) AS dropout_rate
FROM visitor_source_records v
LEFT JOIN (SELECT DISTINCT player_id FROM game_sessions) gs ON gs.player_id = v.player_id
LEFT JOIN (SELECT DISTINCT player_id FROM player_statistics WHERE total_games > 0) ps ON ps.player_id = v.player_id
GROUP BY v.channel, v.scene
ORDER BY start_rate DESC;
```
