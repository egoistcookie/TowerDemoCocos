## 触摸事件传播规则

### 事件监听器分布

| 节点 | 监听器 | 事件类型 | 说明 |
|------|--------|----------|------|
| 单位节点 (Role) | `onTowerClick` | `TOUCH_END` (冒泡阶段) | 处理单位点击，首次点击选中，再次点击移动（添加于 2026/04/08：首次选中时也检测碰撞半径，修复去掉 trim 后贴图覆盖区域变大的问题） |
| 建筑物节点 (Build) | `onBuildingClick` | `TOUCH_END` (目标阶段) | 处理建筑物点击，单击打开信息面板，长按进入拖拽模式（2026/04/08 修复：添加 `_towerBuilderHandlingClick` 标志，防止 TowerBuilder 与建筑物点击冲突） |
| Canvas | `UnitSelectionManager.onGlobalTouchEnd` | `TOUCH_END` (冒泡阶段) | 处理全局点击，用于取消选择（2026/04/08 修复：添加建筑物检查，避免清除建筑物的选择状态） |
| Canvas | `Role.globalTouchHandler` | `TOUCH_END` (冒泡阶段) | 处理选中单位的移动命令（动态注册/移除） |
| Canvas | `SelectionManager.globalTouchHandler` | `TOUCH_END` (冒泡阶段) | 处理框选单位的移动命令（动态注册/移除） |
| Canvas | `TowerBuilder.onTouchStart` | `TOUCH_START` (目标阶段) | 处理建筑物长按检测，设置 `_towerBuilderHandlingClick` 标志 |
| Canvas | `TowerBuilder.onTouchEnd` | `TOUCH_END` (目标阶段) | 处理建筑物单击/长按结束，打开信息面板或进入拖拽模式 |

### 事件执行流程

#### 场景 0：选中 a 单位后，点击 b 单位身侧（不在 b 的选中区域内）【2026/04/09 修复】

```
触摸开始 → b 单位节点 onTowerClick 执行
         → 保存当前选中状态：previouslySelectedUnit = a 单位
         → 检查点击是否在 b 单位的选中区域内 (2 倍宽 3 倍高)
         → 如果不在选中区域内：
           - 检测到 previouslySelectedUnit = a 单位（有其他单位被选中）
           - 获取 a 单位的 Role 组件
           - 调用 a 单位的 setManualMoveTargetPosition(worldPos)
           - return，不阻止事件冒泡
         → 事件传播结束，a 单位移动到点击位置
```

**关键逻辑**：
- 在 `onTowerClick` 方法开始时保存当前选中状态，因为 `UnitSelectionManager` 的状态可能已被更新
- 如果点击不在当前单位的选中区域内，但有其他单位被选中，直接控制已选中单位移动
- 多选情况下，不阻止事件冒泡，让 `SelectionManager.globalTouchHandler` 处理

#### 场景 1：首次点击单位（选中）

```
触摸开始 → 单位节点 onTowerClick 执行
         → 检查点击是否在碰撞半径内（2026/04/08 新增）
         → 如果在碰撞体积外：return，不响应选中
         → 如果在碰撞体积内：设置 event.propagationStopped = true（阻止冒泡）
         → 调用 showUnitInfoPanel()
         → 注册 Role.globalTouchHandler 到 Canvas（延迟 0.1 秒）
         → 事件传播结束
```

#### 场景 2：已选中单位，再次点击单位身旁空地（移动）【2026/04/09 前】

```
触摸开始 → 单位节点 onTowerClick 执行
         → 检测到单位已被选中 (isUnitSelected = true)
         → 调用 setManualMoveTarget(event)
         → 检查点击是否在碰撞体积内：
           - 如果在体积内：return，不移动
           - 如果在体积外：设置移动目标
         → 设置 event.propagationStopped = true
         → 调用 hideSelectionPanel() 清除选择
         → 事件传播结束
```

**注意**：2026/04/09 修复后，此场景的行为改为：点击已选中单位身旁空地时，由 `onTowerClick` 中的"不在选中区域内"逻辑处理，直接调用 `setManualMoveTarget(event)`。

#### 场景 3：已选中单位，点击单位碰撞体积内

```
触摸开始 → 单位节点 onTowerClick 执行
         → 检测到单位已被选中
         → 调用 setManualMoveTarget(event)
         → 检测到点击在碰撞体积内，return
         → 不阻止事件传播
         → Canvas 监听器继续执行（但不会触发移动）
```

#### 场景 4：单击建筑物（选中建筑物）【2026/04/08 修复后】

```
触摸开始 → TowerBuilder.onTouchStart 执行
         → 检测是否点击在建筑物上
         → 如果是：开始长按检测（0.5 秒后显示读条）
         → 设置 (building)._towerBuilderHandlingClick = true
         → 注意：不阻止事件传播，让建筑物的 onBuildingClick 能收到事件
         
触摸结束 → TowerBuilder.onTouchEnd 执行
         → 检查 isLongPressActive = false（未达到长按阈值）
         → 调用 showBuildingInfoPanel(building)
         → building.showSelectionPanel() → UnitSelectionManager.selectUnit()
         → 设置 (building)._showingInfoPanel = true（防止 onBuildingClick 干扰）
         → scheduleOnce(0.5 秒) 后清除 _showingInfoPanel 标志
         
         → 事件传播到建筑物的 onBuildingClick
         → 检查 _towerBuilderHandlingClick = true，跳过处理（返回）
         
         → 事件传播到 UnitSelectionManager.onGlobalTouchEnd
         → 检测到点击的是建筑物（有 Build 组件）
         → 跳过 clearSelection()，保持面板显示
```

#### 场景 5：长按建筑物（进入拖拽模式）【2026/04/08 修复后】

```
触摸开始 → TowerBuilder.onTouchStart 执行
         → 检测是否点击在建筑物上
         → 如果是：开始长按检测
         → 设置 (building)._towerBuilderHandlingClick = true
         
0.5 秒后  → scheduleOnce 回调执行
         → 显示读条指示器（圆环进度条）
         
长按中   → TowerBuilder.checkLongPress 定时器执行（每 0.05 秒）
         → 更新读条进度（0% → 100%）
         
0.8 秒后  → 进度达到 1.0
         → 调用 startDraggingBuilding()
         → 隐藏读条指示器
         → 进入拖拽模式（isDraggingBuilding = true）
         
触摸结束 → TowerBuilder.onTouchEnd 执行
         → 检测到 isDraggingBuilding = true
         → 调用 endDraggingBuilding() 处理拖拽结束
         → 尝试放置建筑物到新位置
         → 事件传播结束
```

#### 场景 6：建筑物上已有信息面板，再次点击

```
触摸开始 → TowerBuilder.onTouchStart 执行
         → 设置 (building)._towerBuilderHandlingClick = true
         
触摸结束 → TowerBuilder.onTouchEnd 执行
         → 调用 showBuildingInfoPanel(building)
         
         → 事件传播到建筑物的 onBuildingClick
         → 检查 _showingInfoPanel = true，跳过处理（返回）
         
         → 事件传播到 UnitSelectionManager.onGlobalTouchEnd
         → 检测到点击的是建筑物，跳过 clearSelection()
         → 面板保持显示
```

### 碰撞体积检测

单位使用 `collisionRadius` 属性定义碰撞体积（世界坐标像素）：
- 弓箭手 (Arrower): 10 像素
- 女猎手 (Hunter): 10 像素
- 精灵剑士 (ElfSwordsman): 10 像素
- 其他单位：查看各自组件的 `collisionRadius` 属性

**屏幕坐标转换**：
由于相机可能有缩放，世界坐标的像素值不等于屏幕坐标的像素值。
检测方法：
1. 获取单位世界坐标 → 转换为屏幕坐标
2. 获取单位 + 碰撞半径位置 → 转换为屏幕坐标
3. 计算两个屏幕位置的距离 = 屏幕坐标下的碰撞半径
4. 计算点击位置到单位中心的距离
5. 如果 距离 <= 碰撞半径，则认为点击在碰撞体积内

## 性能优化分析

### 当前实现的性能特点

#### 优点

1. **事件监听器动态注册/移除**
   - `Role.globalTouchHandler` 仅在单位被选中时注册
   - 移动完成后立即移除，避免无效的每帧检查

2. **碰撞检测频率控制**
   - `Role` 类中有 `COLLISION_CHECK_INTERVAL = 0.05`（每 0.05 秒检测一次）
   - 避免每帧进行昂贵的碰撞检测

3. **Vec3 对象复用**
   - `Role` 类中定义了 `tempVec3_1`, `tempVec3_2`, `tempVec3_3` 临时变量
   - 减少垃圾回收压力

4. **Camera 引用缓存** ✅（2026/04/08）
   - `Role` 类中定义了 `cachedCamera` 属性，在 `start()` 中初始化
   - `setManualMoveTarget` 方法使用缓存的 Camera，避免每次点击都调用 `find('Canvas/Camera')`
   - 减少场景树遍历，提升点击响应速度

5. **建筑物点击事件优化** ✅（2026/04/08）
   - 使用 `_towerBuilderHandlingClick` 标志协调 TowerBuilder 与建筑物的点击处理
   - `UnitSelectionManager.onGlobalTouchEnd` 检测建筑物时跳过清除选择
   - 避免了面板闪烁和误清除问题

#### 可优化点

1. **屏幕坐标转换重复计算** ⚠️ 已分析，暂不优化

   **问题**：`setManualMoveTarget` 中每次都进行两次 `worldToScreen` 转换：
   ```typescript
   cam.worldToScreen(unitWorldPos, unitScreenPos);
   cam.worldToScreen(testWorldPos, testScreenPos);
   ```

   **优化方案**：缓存单位的屏幕坐标，在单位位置变化时更新，而不是每次点击都转换。
   
   **预期收益**：减少矩阵运算，提升点击响应速度。
   
   **暂不优化原因**：
   - 点击操作频率较低（用户每秒点击次数有限），矩阵运算开销不大
   - 单位位置可能随时变化（移动、动画），缓存屏幕坐标需要频繁更新，复杂度增加
   - 通过提取公共方法，代码已更清晰，后续如需统一优化只需修改一处

2. **`isTouchOnUnit` 方法可复用** ✅ 已完成（2026/04/08）

   **问题**：`UnitSelectionManager.isTouchOnUnit` 和 `Role.setManualMoveTarget` 中有重复的碰撞检测逻辑。

   **优化方案**：提取为工具函数或添加到 `Role` 类作为公共方法：
   ```typescript
   // 在 Role.ts 中添加
   isPointInCollisionRadius(touchLocation: Vec3, camera: Camera): boolean {
       // ... 检测逻辑
   }
   ```

   **预期收益**：代码复用，减少维护成本。

   **实现状态**：已在 `Role.ts` 中实现 `isPointInCollisionRadius` 方法，`onTowerClick`、`setManualMoveTarget` 和 `UnitSelectionManager.isTouchOnUnit` 均调用此公共方法。

3. **`scheduleOnce` 延迟清除选择** ⚠️ 已分析，暂不优化

   **问题**：`UnitSelectionManager.onGlobalTouchEnd` 使用 `scheduleOnce(() => { this.clearSelection(); }, 0.01)` 延迟清除，但实际可能不需要延迟。

   **优化方案**：如果确认事件传播顺序稳定，可以同步清除选择：
   ```typescript
   // 在 onGlobalTouchEnd 中直接调用
   this.clearSelection();
   ```

   **预期收益**：减少一帧延迟，响应更即时。
   
   **暂不优化原因**：
   - 延迟清除是为了让单位的 `globalTouchHandler` 先处理移动命令
   - 如果同步清除，可能导致移动命令无法正确响应

4. **单位选中状态检查冗余** ⚠️ 已分析，暂不优化

   **问题**：多个监听器中重复检查 `unitSelectionManager.isUnitSelected(this.node)`。

   **优化方案**：使用标志位缓存选中状态：
   ```typescript
   private isCurrentSelected: boolean = false;
   
   // 在 selectUnit/clearSelection 时更新
   ```

   **预期收益**：减少跨组件调用。
   
   **暂不优化原因**：
   - 增加状态同步的复杂度
   - 当前检查开销很小，性能影响可忽略

5. **建筑物点击事件优化** ✅ 已完成（2026/04/08）

   **问题**：TowerBuilder 与建筑物的点击处理存在竞争条件，导致面板闪烁或无法显示。

   **已实施方案**：
   - `TowerBuilder.onTouchStart` 设置 `_towerBuilderHandlingClick` 标志
   - `Build.onBuildingClick` 检查标志，如果 TowerBuilder 正在处理则跳过
   - `UnitSelectionManager.onGlobalTouchEnd` 检测建筑物时跳过清除选择

   **预期收益**：建筑物点击响应稳定，面板显示正常。

6. **Component 缓存优化** ⚠️ 可考虑

   **问题**：`Build.showSelectionPanel` 等方法中多次调用 `getComponent` 获取组件。

   **优化方案**：在 `start()` 或 `onEnable()` 中缓存常用组件引用：
   ```typescript
   private cachedWarAncientTree: WarAncientTree | null = null;
   
   start() {
       this.cachedWarAncientTree = this.node.getComponent(WarAncientTree);
   }
   ```

   **预期收益**：减少组件查找开销。
   
   **暂不优化原因**：
   - 建筑物点击频率低，组件查找开销可忽略
   - 增加缓存管理复杂度，需要考虑组件移除/复用的情况

### 性能监控建议

1. **使用 Cocos Creator 性能分析器**
   - 开启 `Profiler` 查看帧率
   - 关注 `Logic` 和 `Render` 耗时

2. **添加性能日志（仅开发环境）**
   ```typescript

       const start = performance.now();

       console.log(`Collision check: ${(performance.now() - start).toFixed(2)}ms`);
   ```

3. **压力测试**

   - 频繁点击移动（每秒 5-10 次）
   - 观察帧率是否稳定在 60 FPS

## 调试指南

### 问题：点击单位没有反应

**排查步骤**：
1. 检查 `onTowerClick` 是否被调用（添加临时日志）
2. 检查 `unitSelectionManager.isUnitSelected()` 返回值
3. 检查 `setManualMoveTarget` 中的碰撞检测结果
4. 检查 `cachedCamera` 是否正确初始化

### 问题：点击后单位不移动

**排查步骤**：
1. 检查点击位置是否在碰撞体积内（日志显示 `distance` 和 `collisionRadiusScreen`）
2. 检查 `manualMoveTarget` 是否被正确设置
3. 检查单位是否处于寻敌/攻击状态（可能覆盖移动目标）
4. 检查 `globalTouchHandler` 是否已注册到 Canvas

### 问题：取消选择失效

**排查步骤**：
1. 检查 `UnitSelectionManager.onGlobalTouchEnd` 是否执行
2. 检查 `clearSelection` 是否被调用
3. 检查是否有多个监听器同时修改选中状态
4. 检查 `globalTouchHandler` 是否正确清除

### 问题：建筑物信息面板不显示/闪烁

**排查步骤**：
1. 检查 `TowerBuilder.onTouchEnd` 是否调用 `showBuildingInfoPanel`
2. 检查 `showBuildingInfoPanel` 中建筑物组件是否正确获取
3. 检查 `Build.showSelectionPanel` 是否被调用
4. 检查 `UnitSelectionManager.selectUnit` 是否执行
5. 检查 `onGlobalTouchEnd` 是否跳过建筑物的清除选择
6. 检查 `_showingInfoPanel` 标志是否正确设置和清除
7. 检查 `_towerBuilderHandlingClick` 标志的时序

### 问题：长按建筑物不显示读条

**排查步骤**：
1. 检查 `startLongPressDetection` 是否被调用
2. 检查 `scheduleOnce(0.5)` 回调是否执行
3. 检查 `showLongPressIndicator` 是否被调用
4. 检查建筑物节点是否有效（`isValid === true`）
5. 检查 `longPressIndicator` 是否正确创建并设置父节点
6. 检查 `checkLongPress` 定时器是否正常运行
7. 确认 `longPressThreshold` 的值（当前为 0.8 秒）
