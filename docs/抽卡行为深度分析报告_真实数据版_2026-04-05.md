## 二、广告抽卡比率下降原因分析

### 2.1 时间线对照

| 日期 | 改动内容 | 广告抽卡比率 | 变化 |
|------|---------|-------------|------|
| 03-30 | 增加 SP 卡片类型 | 17.33% | - |
| 03-31 | - | 10.09% | ↓ 7.24pp |
| 04-01 | 石墙/哨塔/弓箭手增加 SP | 15.47% | ↑ 5.38pp |
| 04-02 | 防御塔增加 SP、再抽一次必得 UR+SP | 23.01% | ↑ 7.54pp |
| **04-03** | **前四关首抽固定三张 SP** | **8.81%** | **↓ 14.20pp** |
| 04-04 | - | 3.83% | ↓ 4.98pp |
| 04-05 | - | 1.93% | ↓ 1.90pp |

### 2.2 核心原因确认

**04-03 的"首抽固定三张 SP"改动是广告抽卡比率暴跌的直接原因：**
- 改动前 (04-02)：23.01%
- 改动后 (04-03)：8.81%
- **下降幅度：61.9%**

**原因分析：**
1. 固定三张 SP（石墙 + 哨塔 + 弓箭手）都是 T0 级别核心卡
2. 玩家无需看广告 reroll 即可获得强力开局
3. "全部获取"的边际收益大幅降低（三张卡已经有两张核心 SP）

### 2.3 次要原因

1. **SP 卡片价值稀释**
   - SP 卡池从 5 种扩大到 12+ 种
   - 单张 SP 的稀缺性感知下降

2. **UR 卡吸引力不足**
   - UR 选择率仅 9.83%，低于 SP
   - 说明 UR 卡效果与 SP 重叠，缺乏独特性

3. **玩家学习效应**
   - 老玩家已适应固定 SP 开局策略
   - 不再追求"完美开局"，降低重抽意愿

## 三、优化方案建议（基于真实数据）

### 3.1 方案 A：调整首抽 SP 数量（推荐优先级：P0）

**当前问题：** 固定三张 SP 过于强力，玩家无需看广告

**建议方案：** 改为固定 2 张 SP + 1 张随机（SSR/UR 保底）
- 前两张仍为固定 SP（石墙 + 哨塔）
- 第三张从玩家当前已上场单位中随机生成 SSR/UR 卡

**预期效果：**
- 保留固定 SP 的新手引导作用
- 第三张随机卡可能不理想 → 刺激 reroll 欲望
- 基于 04-02 数据（23.01%），预计回升至 15-18%

### 3.2 方案 B：增强 UR 卡吸引力（推荐优先级：P1）

**当前问题：** UR 选择率仅 9.83%，低于 SP 的 30.20%

**建议方案：**
1. UR 卡增加"全局增益"效果（如全队攻击力 +20%）
2. UR 卡边框增加动态光效（视觉差异化）
3. 在抽卡界面显示 UR 卡片的"质变效果"说明

**预期效果：**
- 提升 reroll 模式的预期价值
- 广告抽卡比率预计提升 3-5%

### 3.3 方案 C：增加广告触发场景（推荐优先级：P2）

**当前问题：** 广告触发场景单一（仅 reroll/get_all）

**建议方案：**
1. 失败复活（已存在，可增加"第二次复活"选项）
2. 金币翻倍（通关后看广告，金币奖励×2）
3. 临时 buff（本局内攻击力 +50%，持续 2 分钟）
4. 抽卡刷新券（每日 3 次免费刷新，之后看广告）

**预期效果：**
- 广告总曝光量提升 50-100%
- 不直接影响广告抽卡比率，但提升总收入

## 四、结论与行动项

### 4.1 核心结论

1. **首抽固定三张 SP 是广告抽卡比率暴跌的主因**
   - 从 23.01% 降至 1.93%，下降 91.6%
   - 04-03 至 04-05 持续走低，无明显回升迹象

2. **SP 卡片强度获得玩家认可**
   - 选择率 30.20%，与 R/SR 相当
   - 多重箭（弓箭手 SP）是最热门卡片

3. **UR 卡吸引力不足**
   - 选择率仅 9.83%，低于预期
   - 需要差异化设计，避免与 SP 效果重叠

## 附录：SQL 查询语句

### A1. 每日广告抽卡比率

```sql
SELECT 
    DATE(FROM_UNIXTIME(e.event_time / 1000)) AS date,
    COUNT(*) AS total_selections,
    SUM(CASE WHEN e.selection_mode IN ('get_all', 'reroll') THEN 1 ELSE 0 END) AS ad_selections,
    ROUND(SUM(CASE WHEN e.selection_mode IN ('get_all', 'reroll') THEN 1 ELSE 0 END) * 100.0 / COUNT(*), 2) AS ad_rate_percent
FROM card_selection_events e
WHERE e.event_time >= UNIX_TIMESTAMP(DATE_SUB(NOW(), INTERVAL 7 DAY)) * 1000
GROUP BY DATE(FROM_UNIXTIME(e.event_time / 1000))
ORDER BY date;
```

### A2. 各稀有度卡片选择率

```sql
SELECT 
    JSON_UNQUOTE(JSON_EXTRACT(c.card, '$.rarity')) AS rarity,
    COUNT(*) AS pick_count,
    ROUND(COUNT(*) * 100.0 / (SELECT COUNT(*) FROM card_selection_events 
        WHERE event_time >= UNIX_TIMESTAMP(DATE_SUB(NOW(), INTERVAL 7 DAY)) * 1000), 2) AS pick_rate_percent
FROM card_selection_events e,
JSON_TABLE(e.cards_json, '$[*]' COLUMNS(card JSON PATH '$')) AS c
WHERE e.event_time >= UNIX_TIMESTAMP(DATE_SUB(NOW(), INTERVAL 7 DAY)) * 1000
GROUP BY JSON_UNQUOTE(JSON_EXTRACT(c.card, '$.rarity'))
ORDER BY pick_count DESC;
```

### A3. SP 卡片各单位热度

```sql
SELECT 
    JSON_UNQUOTE(JSON_EXTRACT(c.card, '$.unitId')) AS unit_id,
    JSON_UNQUOTE(JSON_EXTRACT(c.card, '$.buffType')) AS buff_type,
    COUNT(*) AS pick_count
FROM card_selection_events e,
JSON_TABLE(e.cards_json, '$[*]' COLUMNS(card JSON PATH '$')) AS c
WHERE JSON_UNQUOTE(JSON_EXTRACT(c.card, '$.rarity')) = 'SP'
    AND e.event_time >= UNIX_TIMESTAMP(DATE_SUB(NOW(), INTERVAL 7 DAY)) * 1000
GROUP BY JSON_UNQUOTE(JSON_EXTRACT(c.card, '$.unitId')), JSON_UNQUOTE(JSON_EXTRACT(c.card, '$.buffType'))
ORDER BY pick_count DESC
LIMIT 15;
```

### A4. 首抽 vs 后续抽卡对比

```sql
WITH first_pick AS (
    SELECT 
        e.session_id,
        MIN(e.event_time) AS first_event_time
    FROM card_selection_events e
    WHERE e.event_time >= UNIX_TIMESTAMP(DATE_SUB(NOW(), INTERVAL 7 DAY)) * 1000
    GROUP BY e.session_id
),
pick_analysis AS (
    SELECT 
        e.session_id,
        e.event_time,
        e.selection_mode,
        CASE WHEN e.event_time = fp.first_event_time THEN 'first' ELSE 'subsequent' END AS pick_type
    FROM card_selection_events e
    JOIN first_pick fp ON e.session_id = fp.session_id
    WHERE e.event_time >= UNIX_TIMESTAMP(DATE_SUB(NOW(), INTERVAL 7 DAY)) * 1000
)
SELECT 
    pick_type,
    COUNT(*) AS total,
    SUM(CASE WHEN selection_mode IN ('get_all', 'reroll') THEN 1 ELSE 0 END) AS ad_count,
    ROUND(SUM(CASE WHEN selection_mode IN ('get_all', 'reroll') THEN 1 ELSE 0 END) * 100.0 / COUNT(*), 2) AS ad_rate
FROM pick_analysis
GROUP BY pick_type;
```
