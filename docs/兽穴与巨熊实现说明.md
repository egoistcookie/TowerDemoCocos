## 二、预制体创建指南

### 2.1 兽穴预制体（BeastDen.prefab）

**创建步骤：**

1. 在 `assets/resources/prefabs/` 目录下创建 `BeastDen.prefab`
2. 节点结构：
   ```
   BeastDen (Prefab)
   └── Sprite (Sprite 组件)
       └── 绑定 BeastDen.ts 脚本
   ```

**组件配置：**

| 组件 | 属性 | 值 |
|-----|------|-----|
| Sprite | spriteFrame | 兽穴图片 |
| UITransform | Content Size | 根据图片尺寸 |
| BeastDen (脚本) | triggerRange | 100 |
| BeastDen (脚本) | bearSpawnDistance | 400 |
| BeastDen (脚本) | warningExclamationInterval | 2.0 |
| BeastDen (脚本) | maxExclamationCount | 3 |
| BeastDen (脚本) | tameDuration | 10.0 |

**图片要求：**
- 兽穴外观：中性/野性风格的洞穴入口
- 建议尺寸：100x100 像素

## 四、游戏流程说明

### 4.1 兽穴触发流程

```
1. 游戏开始 → spawnBeastDen() → 兽穴出现在 (50, 400)
2. 敌我单位靠近 (100 像素内) → 兽穴高亮 + 红边显示
3. 上方出现红色感叹号！
4. 每 2 秒增加 1 个感叹号
5. 达到 3 个感叹号 → 巨熊虚影逐渐清晰 (2 秒渐入)
6. 虚影完全显示后 2 秒 → 生成真实巨熊
```

### 4.2 巨熊行为流程

```
初始状态（中立）:
- 攻击范围内最近的单位（敌我都攻击）
- 不离开兽穴超过 400 像素

巨熊被我方击败:
→ 兽穴转为归顺模式（绿边）
→ 上方出现 10 秒读条
→ 读条结束：巨熊满血复活并归顺我方
→ 归顺巨熊只攻击敌人
→ 超过 400 像素自动返回兽穴

归顺巨熊被击败:
→ 兽穴重置为中立状态
→ 我方单位靠近会再次触发感叹号警告
→ 需要再次击败巨熊才能归顺
```

## 六、配置参数说明

### 6.1 兽穴配置（BeastDen.ts）

| 参数 | 默认值 | 说明 |
|-----|--------|------|
| triggerRange | 100 | 触发范围（像素） |
| bearSpawnDistance | 400 | 巨熊返回判定距离 |
| warningExclamationInterval | 2.0 | 感叹号生成间隔（秒） |
| maxExclamationCount | 3 | 最大感叹号数量 |
| tameDuration | 10.0 | 归顺读条时间（秒） |

### 6.2 巨熊配置（Bear.ts）

| 参数 | 默认值 | 说明 |
|-----|--------|------|
| maxHealth | 1000 | 血量（肉盾单位） |
| attackDamage | 30 | 攻击力 |
| attackRange | 60 | 近战攻击范围 |
| attackInterval | 1.5 | 攻击间隔（秒） |
| moveSpeed | 80 | 移动速度（像素/秒） |
| returnDistance | 400 | 返回判定距离（像素） |

## 八、性能优化建议

1. **巨熊目标查找**：已实现 0.5 秒间隔查找，避免每帧查找
2. **感叹号节点管理**：离开触发范围时清理感叹号
3. **虚影节点复用**：虚影节点创建后隐藏，重复使用
4. **对象池建议**：建议后续将巨熊纳入对象池管理
