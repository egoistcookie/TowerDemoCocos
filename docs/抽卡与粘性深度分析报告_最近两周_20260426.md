## 1. 核心发现（先看结论）

### 1.1 两周整体趋势（可比口径）稳中有升

- 在“至少一抽 + 正常结束”的统一可比口径下，本期胜率 `48.66%`，较上一份报告区间 `45.99%` 提升 `+2.67pp`。  
- 但过程指标呈“更短局时 + 更低击杀”的结构：平均防御时长 `273.10s`（-15.62s），平均击杀 `339.81`（-13.82），说明当前更像是“快节奏分化”而不是“纯强度线性抬升”。

### 1.2 0422 埋点升级后，主要问题确认在“首抽前流失”

- 0422 之后（按新埋点）全会话 `1198`，至少一抽仅 `313`（`26.13%`），正常结束 `101`（`8.43%`）。  
- 至少一抽且正常结束 `93`（`7.76%`）。  
- 结论：当前最大漏斗损耗在“开局 -> 首抽”，不是“已抽卡玩家后段对局”。

### 1.3 最近5天版本波动明显，`1.0.76` 为峰值，`1.0.77` 出现回调

- `1.0.76`（04-25）胜率/时长/击杀/操作均达到近期高位（`65.79%`、`344.53s`、`463.61`、`72.97`）。  
- `1.0.77`（04-26）上线后回落至 `51.22%`、`234.85s`、`310.88`、`47.63`，但同时 `single_video` 由 0 提升到 17，说明新抽卡机制开始起效，短期需要再观察 2-3 天判断是“正常冷却”还是“体验摩擦”。

## 5. 重点问题诊断（结合 0422 分界）

### 5.1 为什么不能直接拿 0422 前后全会话对比

- `2026-04-22 23:29:27` 后，`game_sessions` 开始保存开局 0-12 秒会话；之前无这部分数据。  
- 直接对比“全会话胜率/完成率”会把新增的短会话流失混入，导致前后口径失真。  
- 因此跨分界只能比较“至少一抽 + 正常结束”的可比子集。

### 5.2 当前增长瓶颈

- 0422 后的真实漏斗显示，首抽前流失量级远高于后段流失。  
- 这与 1.0.77 增加抽卡复杂度（稀有度、金币门槛、视频入口）在同一时间段出现，说明新手前 60~90 秒的引导和反馈节奏仍需优化。

## 7. 清晰口径（与需求一致）

1. **非测试记录（会话层）**
   - `player_id not in ('player_1772462826043_800','player_1772466497770_5671','player_1772530937065_3381','player_1772722064044_978','player_1772465771074_4106')`
   - `player_id not in (select player_id from visitor_source_records where channel='unknown')`

2. **至少一抽（会话层）**
   - `id in (select session_id from card_selection_events where game_time != 0)`

3. **正常结束（会话层）**
   - `result is not null`

4. **0422 分界可比原则（关键）**
   - 分界时间：`2026-04-22 23:29:27`
   - 分界前 `game_sessions` 不含 0~12 秒会话；分界后包含。
   - 跨分界对比时，只能用“至少一抽 + 正常结束”口径。

5. **广告抽卡定义**
   - `selection_mode in ('get_all','reroll','single_video')`

6. **选卡统计定义**
   - 取 `operations_json` 中 `select_buff_card`；
   - 优先读取 `details` 子节点：`mode + unitId + rarity + buffType` 作为 key；
   - 若 `details` 缺失，兜底读取后一个 JSON 节点。
