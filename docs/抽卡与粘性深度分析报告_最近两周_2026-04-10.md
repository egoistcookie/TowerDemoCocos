## 七、问题诊断与优化建议

### 7.1 核心问题诊断

#### 问题 1：广告抽卡比率持续低迷（优先级：P0）

**现状**：广告抽卡比率从峰值 23.01% 跌至 4.49%，远低于目标值 15-20%

**根因分析**：
1. **SP 卡片价值稀释**：SP 卡池从 5 种扩大到 12+ 种，单张 SP 稀缺性感知下降
2. **首抽策略固化**：玩家已形成"首抽直接选"的认知，reroll 意愿低
3. **金矿机制分流**：玩家注意力转向资源运营，降低对卡片强度的追求
4. **UR 吸引力不足**：UR 选择率仅 3.95%，缺乏独特价值

#### 问题 2：UR 卡片边缘化（优先级：P1）

**现状**：UR 选择率从 9.83% 降至 3.95%，远低于 SP 的 19.88%

**根因分析**：
1. UR 效果与 SP 重叠，缺乏差异化
2. UR 卡片缺少视觉/数值上的"质变感"
3. reroll 机制虽然保证 UR+SP 同出，但玩家主动选择意愿低

#### 问题 3：女猎手单位持续低迷（优先级：P2）

**现状**：女猎手选择率仅 2.01%，SP 弹弹乐仅 1.8%

**根因分析**：
1. 女猎手作为近战单位，生存能力不足
2. 弹弹乐技能效果不够直观
3. 哨塔/弓箭手/石墙"铁三角"过于强势，挤压其他单位空间

### 7.2 优化方案建议

#### 方案 A：重构首抽机制（推荐优先级：P0）

**建议**：改为"2 张固定 SP + 1 张随机 SSR/UR"
- 前两张仍为固定 SP（石墙自愈 + 哨塔巨弩）
- 第三张从玩家当前已上场单位中随机生成 SSR/UR 卡

**预期效果**：
- 保留固定 SP 的新手引导作用
- 第三张随机卡可能不理想 → 刺激 reroll 欲望
- 基于 04-02 数据（23.01%），预计回升至 12-15%

#### 方案 B：增强 UR 卡差异化（推荐优先级：P1）

**建议**：
1. UR 卡增加"全局增益"效果（如全队攻击力 +20%）
2. UR 卡边框增加动态光效（视觉差异化）
3. 在抽卡界面显示 UR 卡片的"质变效果"说明

**预期效果**：
- 提升 reroll 模式的预期价值
- 广告抽卡比率预计提升 2-3%

#### 方案 C：增加广告触发场景（推荐优先级：P2）

**建议**：
1. 失败复活（已存在，可增加"第二次复活"选项）
2. 金币翻倍（通关后看广告，金币奖励×2）
3. 临时 buff（本局内攻击力 +50%，持续 2 分钟）
4. 抽卡刷新券（每日 3 次免费刷新，之后看广告）

**预期效果**：
- 广告总曝光量提升 50-100%
- 不直接影响广告抽卡比率，但提升总收入

#### 方案 D：优化金矿与抽卡平衡（推荐优先级：P2）

**建议**：
1. 金矿产出金币部分用于抽卡（如每次抽卡消耗 10 金币，可从金矿产出扣除）
2. 增加"金币换抽卡券"功能（100 金币=1 次免费抽卡，每日限 3 次）
3. 金矿被占领时，触发弓箭手/女猎手口号系统（已实现部分）

**预期效果**：
- 将金矿流量导流至抽卡系统
- 提升抽卡频率，间接提升广告使用率

1. **广告抽卡比率暴跌 63.9%**：从 12.44% 降至 4.49%，主要受"首抽固定 SP"机制后遗症影响

4. **金矿机制效果显著**：平均金币提升 68.8%，防御时长提升 19.8%
5. **UR 卡片严重边缘化**：选择率从 9.83% 降至 3.95%，需差异化设计
6. **reroll 机制技术正确但使用率低**：UR+SP 必得机制已实现，但 reroll 使用率仅 3.33%

## 附录：SQL 查询语句

### A1. 每日广告抽卡比率（14 天）

```sql
SELECT 
    DATE(FROM_UNIXTIME(event_time / 1000)) AS date,
    COUNT(*) AS total_selections,
    SUM(CASE WHEN selection_mode IN ('get_all', 'reroll') THEN 1 ELSE 0 END) AS ad_selections,
    ROUND(SUM(CASE WHEN selection_mode IN ('get_all', 'reroll') THEN 1 ELSE 0 END) * 100.0 / COUNT(*), 2) AS ad_rate_percent
FROM card_selection_events 
WHERE event_time >= UNIX_TIMESTAMP(DATE_SUB(NOW(), INTERVAL 14 DAY)) * 1000
GROUP BY DATE(FROM_UNIXTIME(event_time / 1000))
ORDER BY date;
```

### A2. 两周对局趋势总览

```sql
SELECT 
    CASE 
        WHEN end_time >= UNIX_TIMESTAMP(DATE_SUB(NOW(), INTERVAL 7 DAY)) * 1000 THEN '最近 7 天'
        ELSE '前 7 天'
    END AS period,
    COUNT(*) AS total_games,
    SUM(CASE WHEN result = 'success' THEN 1 ELSE 0 END) AS success_games,
    ROUND(SUM(CASE WHEN result = 'success' THEN 1 ELSE 0 END) * 100.0 / COUNT(*), 2) AS win_rate,
    ROUND(AVG(defend_time), 2) AS avg_defend_time,
    ROUND(AVG(kill_count), 2) AS avg_kills,
    ROUND(AVG(operation_count), 2) AS avg_operations
FROM game_records 
WHERE end_time >= UNIX_TIMESTAMP(DATE_SUB(NOW(), INTERVAL 14 DAY)) * 1000
GROUP BY period;
```

### A3. 弃局时长分布对比

```sql
WITH session_last_pick AS (
    SELECT 
        session_id,
        MAX(event_time) AS last_time,
        MAX(game_time) AS last_game_time
    FROM card_selection_events
    WHERE event_time >= UNIX_TIMESTAMP(DATE_SUB(NOW(), INTERVAL 14 DAY)) * 1000
    GROUP BY session_id
),
pick_classified AS (
    SELECT 
        session_id,
        last_game_time,
        CASE 
            WHEN last_time >= UNIX_TIMESTAMP(DATE_SUB(NOW(), INTERVAL 7 DAY)) * 1000 THEN 'recent'
            ELSE 'previous'
        END AS period
    FROM session_last_pick
)
SELECT 
    period,
    COUNT(*) AS total_sessions,
    SUM(CASE WHEN last_game_time <= 30 THEN 1 ELSE 0 END) AS quit_0_30,
    ROUND(SUM(CASE WHEN last_game_time <= 30 THEN 1 ELSE 0 END) * 100.0 / COUNT(*), 2) AS quit_0_30_pct,
    SUM(CASE WHEN last_game_time BETWEEN 31 AND 60 THEN 1 ELSE 0 END) AS quit_31_60,
    ROUND(SUM(CASE WHEN last_game_time BETWEEN 31 AND 60 THEN 1 ELSE 0 END) * 100.0 / COUNT(*), 2) AS quit_31_60_pct,
    SUM(CASE WHEN last_game_time > 180 THEN 1 ELSE 0 END) AS quit_180_plus,
    ROUND(SUM(CASE WHEN last_game_time > 180 THEN 1 ELSE 0 END) * 100.0 / COUNT(*), 2) AS quit_180_plus_pct
FROM pick_classified
GROUP BY period;
```
