# Role.ts 缓存向量机制说明

## 问题背景（2026-04-12 修复）

### 问题现象
`tempVec3_3` 的值在运行中变成了 `(-0.998, 0.05)`，导致弓箭手位置计算错误，出现传送回原点的 bug。

### 根本原因
**临时向量引用冲突**：在 `moveTowardsTarget` 函数中，`towerPos` 被设置为 `tempVec3_3` 的引用（而非拷贝），然后在调用 `calculateAvoidanceDirection` 时，该函数内部修改了 `tempVec3_3`，导致 `towerPos` 的值也被意外修改。

```typescript
// ❌ 错误的写法（修复前）
const towerPos = this.tempVec3_3.set(currentPosX, currentPosY, 0);  // towerPos 是 tempVec3_3 的引用
finalDirection = this.calculateAvoidanceDirection(towerPos, ...);   // 函数内修改了 tempVec3_3
// 此时 towerPos 的值也被修改了！

// ✅ 正确的写法（修复后）
const towerPos = new Vec3(currentPosX, currentPosY, 0);  // 创建新的 Vec3 对象
finalDirection = this.calculateAvoidanceDirection(towerPos, ...);
// towerPos 保持不变
```

### 修复方案
1. 将 `towerPos` 改为使用 `new Vec3()` 创建独立对象

## 编码规范（重要！）

### ⚠️ 临时向量使用陷阱

1. **不要将临时向量的引用赋值给变量后跨函数调用使用**
   ```typescript
   // ❌ 错误：变量持有临时向量引用，被其他函数修改
   const towerPos = this.tempVec3_3.set(x, y, 0);
   someFunction(towerPos);  // someFunction 可能修改 tempVec3_3
   usePosition(towerPos);   // 此时 towerPos 可能已被修改！
   
   // ✅ 正确：创建独立对象
   const towerPos = new Vec3(x, y, 0);
   someFunction(towerPos);
   usePosition(towerPos);   // towerPos 保持不变
   ```

2. **不要将同一个临时向量同时用于输入和输出参数**
   ```typescript
   // ❌ 错误：tempVec3_3 同时作为 lerp 的目标和后续计算的输入
   Vec3.lerp(this.tempVec3_3, dir1, dir2, t);
   useResult(this.tempVec3_3);  // 可能和预期不一致
   
   // ✅ 正确：使用不同的向量
   const result = new Vec3();
   Vec3.lerp(result, dir1, dir2, t);
   useResult(result);
   ```

3. **临时向量不要跨帧保存**
   ```typescript
   // ❌ 错误：临时向量可能在下帧被其他代码修改
   this.savedVector = this.tempVec3_1;
   
   // ✅ 正确：保存值的拷贝
   this.savedVector = this.tempVec3_1.clone();
   ```

4. **函数间传递临时向量时要格外小心**
   - 如果函数会修改传入的向量，调用方要确保不影响后续逻辑
   - 考虑在函数文档中明确标注是否会修改参数

## 兵营生产单位时的位置设置顺序（重要！）

### 问题背景（2026-04-12 修复）

**问题现象**：复活的单位（从对象池获取的单位）不会自动向上移动。

**根本原因**：兵营生产单位时，原来的代码顺序是：
```typescript
// ❌ 错误的顺序（修复前）
unit.active = false;
unit.setParent(container);
// ... 其他初始化 ...
unit.active = true;  // 触发 onEnable()
unit.setWorldPosition(spawnPos);  // 此时才设置位置
```

当 `unit.active = true` 执行时，`onEnable()` 被调用，但此时单位的位置还是**上次死亡时的旧位置**（可能 y >= 500）。这导致 `onEnable()` 中的自动上移逻辑检查 `curPos.y >= 500` 时误判，跳过自动上移安排。

**修复方案**：在激活节点前先设置位置
```typescript
// ✅ 正确的顺序（修复后）
unit.active = false;
unit.setParent(container);
unit.setWorldPosition(spawnPos);  // 先设置位置
// ... 其他初始化 ...
unit.active = true;  // 触发 onEnable()，此时位置已正确
```

### 修复的文件

| 文件 | 修复内容 |
|------|----------|
| `HunterHall.ts` | 在生产猎手时，先设置 `spawnPos` 再激活节点 |
| `SwordsmanHall.ts` | 在生产剑士时，先设置 `spawnPos` 再激活节点 |
| `Church.ts` | 在生产牧师时，先设置 `spawnPos` 再激活节点 |
| `MageTower.ts` | 在生产法师时，先设置 `spawnPos` 再激活节点 |
| `WarAncientTree.ts` | 在生产弓箭手时，先设置 `spawnPos` 再激活节点 |

### 编码规范

**生产/复活单位时的标准流程**：
```typescript
// 1. 从对象池获取单位（此时 unit.active = false）
let unit = UnitPool.getInstance()?.get('UnitType');

// 2. 确保单位处于非激活状态
unit.active = false;

// 3. 设置父节点
unit.setParent(container);

// 4. 【关键】在激活前先设置位置，确保 onEnable 时位置已正确
unit.setWorldPosition(spawnPos);

// 5. 其他初始化（脚本配置、 prefabName 等）
const script = unit.getComponent('Script');
if (script) {
    script.prefabName = 'UnitType';
}

// 6. 激活节点（触发 onEnable()，此时位置已正确）
unit.active = true;
```

## 补充说明：calculateEnemyAvoidanceDirection 的向量陷阱

在 `Enemy.ts` 中，`calculateEnemyAvoidanceDirection` 函数有一个特殊的陷阱：

```typescript
// ❌ 错误的调用方式（修复前）
Vec3.subtract(this.tempVec3_1, targetPos, myPos);  // tempVec3_1 = 移动方向
this.tempVec3_1.normalize();
const avoidanceDir = this.calculateEnemyAvoidanceDirection(currentPos, this.tempVec3_1, deltaTime);
// 函数内部第一行就清空了 tempVec3_1！
// this.flipDirection(this.tempVec3_1);  // 此时 tempVec3_1 = (0,0,0)，不会转向！

// ✅ 正确的调用方式（修复后）
Vec3.subtract(this.tempVec3_1, targetPos, myPos);
this.tempVec3_1.normalize();
// 保存原始方向到独立对象，避免被函数修改
const originalDirection = new Vec3(this.tempVec3_1.x, this.tempVec3_1.y, this.tempVec3_1.z);
const avoidanceDir = this.calculateEnemyAvoidanceDirection(currentPos, originalDirection, deltaTime);
// 使用返回的最终方向
this.tempVec3_1.set(avoidanceDir);
this.flipDirection(this.tempVec3_1);  // 正确转向
```

**原因分析**：`calculateEnemyAvoidanceDirection` 函数内部第一行会清空 `tempVec3_1` 用于累积避让力：
```typescript
this.tempVec3_1.set(0, 0, 0);  // 清空 tempVec3_1
```

如果调用时传入的是 `tempVec3_1`，那么原始方向就会被清空，导致后续转向判断失败。
