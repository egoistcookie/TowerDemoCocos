# 游戏埋点功能说明

## 功能概述

游戏埋点系统已经集成到游戏中，会在游戏结束时自动上报数据到服务器。

## 已实现的功能

### 1. 自动数据收集

游戏会自动收集以下数据：
- 玩家ID（自动生成并保存在本地）
- 关卡数（1-10）
- 游戏结果（成功/失败）
- 防御时间（秒）
- 当前波次
- 最终金币数
- 最终人口数
- 击杀数
- 操作记录（所有玩家操作）

### 2. 数据上报时机

- 游戏胜利时自动上报
- 游戏失败时自动上报
- 上报过程不会阻塞游戏流程（异步执行）

### 3. 服务器配置

服务器地址已配置为：`https://www.egoistcookie.top/api/analytics/report`

## 测试步骤

### 1. 启动游戏

正常启动游戏，系统会自动初始化埋点管理器。

### 2. 开始游戏

点击"开始游戏"按钮，系统会：
- 启动埋点记录
- 记录当前关卡
- 开始记录玩家操作

### 3. 游戏过程

在游戏过程中，所有操作都会被自动记录：
- 建造防御塔
- 训练单位
- 升级建筑
- 回收单位
- 选择增益卡片
- 等等...

### 4. 游戏结束

当游戏结束（胜利或失败）时：
- 系统会自动收集所有数据
- 异步上报到服务器
- 在控制台输出上报结果

### 5. 查看日志

在浏览器控制台（F12）中查看日志：

```
[AnalyticsManager] 开始记录关卡 1
[GameManager] 已启动关卡 1 的埋点记录
[AnalyticsManager] 记录操作: build_watchtower, 游戏时间: 5.2s
...
[GameManager] 准备上报埋点数据: { result: 'success', defendTime: 120, ... }
[AnalyticsManager] 数据上报成功: {...}
[GameManager] 埋点数据上报成功
```

## 服务器端查看数据

### 1. 查询玩家统计

```bash
curl https://www.egoistcookie.top/api/analytics/player/{playerId}
```

### 2. 查询排行榜

```bash
curl https://www.egoistcookie.top/api/analytics/leaderboard?limit=10
```

### 3. 查询关卡统计

```bash
curl https://www.egoistcookie.top/api/analytics/levels
```

## 注意事项

1. **玩家ID**：首次运行时会自动生成，保存在本地存储中
2. **网络要求**：需要能访问服务器 `www.egoistcookie.top`
3. **数据上报**：即使上报失败也不会影响游戏正常运行
4. **操作记录**：目前已支持记录所有主要操作类型

## 操作类型列表

系统支持记录以下操作类型：

- `build_watchtower` - 建造哨塔
- `build_war_ancient_tree` - 建造战争古树
- `build_hunter_hall` - 建造猎手大厅
- `build_swordsman_hall` - 建造剑士小屋
- `build_church` - 建造教堂
- `build_moon_well` - 建造月亮井
- `build_stone_wall` - 建造石墙
- `build_tree` - 建造树木
- `build_thunder_tower` - 建造雷塔
- `build_ice_tower` - 建造冰塔
- `train_arrower` - 训练弓箭手
- `train_hunter` - 训练女猎手
- `train_swordsman` - 训练剑士
- `train_priest` - 训练牧师
- `train_wisp` - 训练小精灵
- `upgrade_unit` - 升级单位
- `recycle_unit` - 回收单位
- `upgrade_building` - 升级建筑
- `recycle_building` - 回收建筑
- `select_buff_card` - 选择增益卡片
- `use_talent_point` - 使用天赋点

## 如何记录自定义操作

如果需要在游戏中记录其他操作，可以使用：

```typescript
import { AnalyticsManager, OperationType } from './AnalyticsManager';

// 获取管理器实例
const analyticsManager = AnalyticsManager.getInstance();

// 记录操作
analyticsManager.recordOperation(
    OperationType.BUILD_WATCHTOWER,  // 操作类型
    this.gameTime,                    // 游戏时间（秒）
    { position: { x: 100, y: 200 } }  // 操作详情（可选）
);
```

## 故障排查

### 1. 数据没有上报

检查控制台是否有错误信息：
- 网络连接是否正常
- 服务器是否正常运行
- 端口是否正确（默认3001）

### 2. 服务器连接失败

确认服务器状态：
```bash
curl https://www.egoistcookie.top/api/health
```

应该返回：
```json
{"status":"ok","timestamp":1234567890}
```

### 3. 数据格式错误

检查控制台日志，确认所有必要字段都已收集。

## 数据隐私

- 玩家ID是随机生成的，不包含任何个人信息
- 只收集游戏内的操作数据
- 数据仅用于游戏分析和优化
