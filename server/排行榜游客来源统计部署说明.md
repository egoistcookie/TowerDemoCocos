# 排行榜游客来源统计功能部署说明

## 功能概述

本次修改新增了游客来源渠道统计功能，用于追踪查看排行榜但未上传游戏记录的游客（约占 97%）的来源渠道和访问场景。

### 改动内容

1. **前端修改** (`assets/scripts/GameManager.ts`)
   - 在 `kill-rank` 请求中增加了 `channel`（来源渠道）和 `scene`（访问场景）参数
   - 从微信小游戏启动参数中自动获取场景值，判断来源渠道

2. **后端修改** (`server/analytics-api.js`)
   - 修改 `/api/analytics/player/:playerId/kill-rank` 接口，支持 `channel` 和 `scene` 查询参数
   - 新增游客来源记录逻辑，写入 `visitor_source_records` 表

3. **数据库新增**
   - 新增表：`visitor_source_records` - 游客来源记录表

## 部署步骤

### 第一步：执行数据库迁移

在 MySQL 中执行以下 SQL 脚本创建新表：

```bash
# 方式 1：使用 mysql 命令行
mysql -u tower_game_user -p'TowerGame@2026!' tower_defense_analytics < database/create_visitor_source_table.sql

# 方式 2：登录 MySQL 后手动执行
mysql -u tower_game_user -p
USE tower_defense_analytics;
source database/create_visitor_source_table.sql;
```

### 第二步：重启后端服务

重启 Node.js 后端服务以应用新的 API 代码：

```bash
# 假设服务使用 PM2 管理
pm2 restart tower-analytics

# 或者直接重启 Node 进程
node server/analytics-api.js
```

### 第三步：重新构建并发布前端

使用 Cocos Creator 重新构建项目并发布到微信小游戏平台。

## 数据查询

### 查看游客来源统计

```sql
-- 各渠道游客数量排名
SELECT channel, COUNT(*) AS visitor_count
FROM visitor_source_records
GROUP BY channel
ORDER BY visitor_count DESC;
```

### 查看游客转化率

```sql
-- 各渠道转化率（有游戏记录的玩家占比）
SELECT
    v.channel,
    COUNT(*) AS total_visitors,
    COUNT(p.player_id) AS converted_players,
    ROUND(COUNT(p.player_id) * 100.0 / COUNT(*), 2) AS conversion_rate
FROM visitor_source_records v
LEFT JOIN player_statistics p ON p.player_id = v.player_id AND p.total_games > 0
GROUP BY v.channel
ORDER BY conversion_rate DESC;
```

## 注意事项

1. **首次部署时必须执行建表 SQL**，否则后端会记录"表不存在"的警告
2. **数据记录不影响排行榜功能**，即使表不存在或写入失败，排行榜查询仍正常工作
3. **隐私保护**：仅记录玩家 ID、来源渠道和场景，不涉及敏感个人信息
4. **数据清理**：建议定期清理长期未访问的游客记录，避免表数据过大
