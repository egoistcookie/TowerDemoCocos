```typescript

const OFFSET_ANGLES = [60, -60, 90, -90, 120, -120, 150, -150, 30, -30];
const OFFSET_COS: number[] = [];
const OFFSET_SIN: number[] = [];
(function() {
    for (const angle of OFFSET_ANGLES) {
        const rad = angle * Math.PI / 180;
        OFFSET_COS.push(Math.cos(rad));
        OFFSET_SIN.push(Math.sin(rad));
    }
})();

// 避让循环
for (let i = 0; i < OFFSET_ANGLES.length; i++) {
    const cosRad = OFFSET_COS[i];  // ✅ 查表
    const sinRad = OFFSET_SIN[i];  // ✅ 查表
    
    // 旋转方向向量
    const offsetDirX = this.tempVec3_1.x * cosRad - this.tempVec3_1.y * sinRad;
```

```typescript
// 属性
private detectionTimer: number = 0;
private readonly DETECTION_INTERVAL: number = 0.5;
private cachedHasUnitsInRange: boolean = false;

update(deltaTime: number) {
    // 每 0.5 秒检测一次范围内单位，降低 CPU 开销
    this.detectionTimer += deltaTime;
    if (this.detectionTimer >= this.DETECTION_INTERVAL) {
        this.detectionTimer = 0;
        
        const myPos = this.node.worldPosition;
        const friendlyCount = this.countFriendliesInRange(myPos, this.detectionRange);

    // 使用缓存的值
    this.hasUnitsInRange = this.cachedHasUnitsInRange;
}
```

#### 2. Enemy.ts 递归查找敌人容器（严重）⭐

**问题描述**：`calculateEnemyAvoidanceDirection` 方法使用递归查找敌人容器。

```typescript

    for (const child of node.children) {
        const found = findNodeRecursive(child, name);
        if (found) return found;
    }
    return null;
};

// 2560-2568
const enemyContainers = ['Enemies', 'Orcs', 'TrollSpearmans', 'OrcWarriors', 'OrcWarlords', 'MinotaurWarriors'];
```

**性能影响**：
- 每次避障计算都可能触发递归查找

- 6 个容器名称，最坏情况下递归遍历整个场景树 6 次

1. 缓存敌人容器节点引用（`cachedEnemyContainers`）

## 📝 2026/04/12 moveTowardsTarget 方法 Vec3 引用冲突修复

### 问题描述

在 `Role.ts` 的 `moveTowardsTarget` 方法中，存在严重的 Vec3 对象引用冲突问题：

**问题代码**（第 1657 行）：
```typescript

```

这里 `towerPos` 是 `tempVec3_1` 的引用（同一个对象），而后续代码中多次修改 `tempVec3_1`：

1. 第 1724 行：`Vec3.scaleAndAdd(this.tempVec3_1, towerPos, pushDirection, pushDistance);`
   - 修改 `tempVec3_1`，也就是修改了 `towerPos`！
   
2. 第 1735 行：`Vec3.subtract(this.tempVec3_1, targetPos, towerPos);`
   - 再次修改 `tempVec3_1`，`towerPos` 再次被修改！

**导致的症状**：
- 单位位置计算错误
- 卡住检测逻辑使用被污染的位置数据
- 单位可能闪现到错误位置

### 修复方案

使用局部变量存储坐标值，避免与 `tempVec3_1/2/3` 引用冲突：


**关键改动**：

2. 需要 `Vec3` 对象时，使用 `tempVec3_3`（与 `tempVec3_1/2` 隔离）
3. 每次使用时重新设置值，避免跨步骤污染

### 预期收益

- 修复单位位置计算错误的问题
- 修复单位闪现到原点的问题

## 📝 临时 Vec3 对象复用编码规范

基于上述修复，总结以下编码规范：

### ⚠️ 临时向量使用陷阱

1. **不要将临时向量的引用赋值给变量后跨函数调用使用**
   ```typescript
   // ❌ 错误：变量持有临时向量引用，被其他函数修改
   const towerPos = this.tempVec3_3.set(x, y, 0);
   someFunction(towerPos);  // someFunction 可能修改 tempVec3_3
   usePosition(towerPos);   // 此时 towerPos 可能已被修改！

   // ✅ 正确：创建独立对象
   const towerPos = new Vec3(x, y, 0);
   someFunction(towerPos);
   usePosition(towerPos);   // towerPos 保持不变
   ```

2. **不要将同一个临时向量同时用于输入和输出参数**
   ```typescript
   // ❌ 错误：tempVec3_3 同时作为 lerp 的目标和后续计算的输入
   Vec3.lerp(this.tempVec3_3, dir1, dir2, t);
   useResult(this.tempVec3_3);  // 可能和预期不一致

   // ✅ 正确：使用不同的向量
   const result = new Vec3();
   Vec3.lerp(result, dir1, dir2, t);
   useResult(result);
   ```

3. **临时向量不要跨帧保存**
   ```typescript
   // ❌ 错误：临时向量可能在下帧被其他代码修改
   this.savedVector = this.tempVec3_1;

   // ✅ 正确：保存值的拷贝
   this.savedVector = this.tempVec3_1.clone();
   ```

4. **函数间传递临时向量时要格外小心**
   - 如果函数会修改传入的向量，调用方要确保不影响后续逻辑
   - 考虑在函数文档中明确标注是否会修改参数

### 📋 兵营生产单位时的标准流程

```typescript
// 1. 从对象池获取单位（此时 unit.active = false）
let unit = UnitPool.getInstance()?.get('UnitType');

// 2. 确保单位处于非激活状态
unit.active = false;

// 3. 设置父节点

// 4. 【关键】在激活前先设置位置，确保 onEnable 时位置已正确

// 5. 其他初始化（脚本配置、prefabName 等）

if (script) {
    script.prefabName = 'UnitType';
}

// 6. 激活节点（触发 onEnable()，此时位置已正确）

```
